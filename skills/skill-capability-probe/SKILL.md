# Skill 能力探测

本技能用于探测智能体运行环境对 SKILL 规范的支持程度，零外部依赖，快速完成。

## 角色与目标

你是一个环境诊断工具。你的任务是按顺序执行以下 8 项能力测试，每项测试完成后立即记录结果，最后输出一份结构化报告。

**执行原则：**
1. 严格按顺序执行，不要跳过任何步骤
2. 每项测试只有"通过"或"失败"两种结果，失败时记录原因
3. 不要因为某项失败而停止后续测试
4. 最终输出必须符合"输出规范"章节的格式

## 第一步：SKILL.md 加载测试

验证智能体是否能读取本 SKILL.md 文件。

**操作：** 复述本文件 frontmatter（`---` 之间的部分）中的以下字段：
- `name`
- `description` 的前 20 个字符
- `metadata.openclaw.env` 中声明的环境变量名

**判定：**
- ✓ 能正确复述以上字段 → 通过
- ✗ 无法读取或字段缺失 → 失败

## 第二步：环境变量读取测试

验证环境变量是否能正确传递到智能体运行环境。

**操作：**
1. 读取环境变量 `PROBE_TEST_VAR` 的值
2. 如果该变量未设置，使用默认值 `probe-default`

**判定：**
- ✓ 能读取到值（无论是否为默认值）→ 通过
- ✗ 读取报错或无法访问环境变量 → 失败

**记录：** 实际读取到的值。

## 第三步：文件写入测试

验证智能体是否有文件系统写入权限。

**操作：**
1. 在当前工作目录下创建文件 `probe-test.txt`
2. 写入以下内容（替换占位符）：
   ```
   probe-timestamp: <当前时间>
   probe-user: <第二步读取的 PROBE_TEST_VAR 值>
   probe-step: file-write
   ```

**判定：**
- ✓ 文件创建成功 → 通过
- ✗ 创建失败（权限不足、路径不存在等）→ 失败

**记录：** 文件的完整路径。

## 第四步：文件读取测试

验证智能体能否读回刚写入的文件。

**操作：**
1. 读取第三步创建的 `probe-test.txt`
2. 验证内容是否包含 `probe-step: file-write`

**判定：**
- ✓ 读取成功且内容匹配 → 通过
- ✗ 读取失败或内容不匹配 → 失败

## 第五步：命令执行测试

验证智能体能否执行 shell 命令。

**操作：** 依次执行以下命令并记录输出：
1. `whoami` — 获取当前用户
2. `pwd` — 获取当前工作目录
3. `python3 --version` 或 `python --version` — 检查 Python 是否可用
4. `node --version` — 检查 Node.js 是否可用

**判定：**
- ✓ 至少能执行 `whoami` 和 `pwd` → 通过
- ✗ 无法执行任何命令 → 失败

**记录：** 每条命令的输出或错误信息。Python/Node.js 的可用性单独记录，不影响本项判定。

## 第六步：多步骤执行测试

验证智能体能否按顺序完成多个子步骤。

**操作：** 按以下顺序执行（顺序不能错）：
1. 创建目录 `probe-dir`
2. 在该目录下创建文件 `step1.txt`，内容为 `step-1`
3. 创建文件 `step2.txt`，内容为 `step-2`
4. 列出 `probe-dir` 目录内容
5. 删除 `step1.txt`
6. 再次列出目录内容，确认只剩 `step2.txt`

**判定：**
- ✓ 6 个子步骤全部按顺序完成，且最终目录只剩 `step2.txt` → 通过
- ✗ 任何步骤失败或顺序错误 → 失败

## 第七步：清理测试文件

验证智能体能否删除文件和目录。

**操作：**
1. 删除 `probe-dir` 目录及其内容
2. 删除 `probe-test.txt`

**判定：**
- ✓ 清理完成，工作目录下无残留测试文件 → 通过
- ✗ 删除失败 → 失败

## 第八步：输出规范化测试

本步骤即输出最终报告。验证智能体能否严格遵循指定的输出格式。

## 输出规范

完成所有测试后，必须输出以下格式的报告（不要添加额外说明）：

```
[Skill 能力探测报告]
==============================
[✓/✗] SKILL.md 加载        — <结果说明>
[✓/✗] 环境变量读取          — PROBE_TEST_VAR=<实际值>
[✓/✗] 文件写入             — <文件路径>
[✓/✗] 文件读取             — <内容匹配情况>
[✓/✗] 命令执行             — <用户>, <工作目录>, Python:<是/否>, Node:<是/否>
[✓/✗] 多步骤执行           — <完成步数>/6
[✓/✗] 清理测试文件          — <清理结果>
[✓/✗] 输出规范化           — 本报告
==============================
结论：<通过数>/8 项通过。<建议>
```

### 建议生成规则

根据通过的项目数，给出对应建议：

- **8/8 通过**：环境完全支持 Skills，可使用所有技能。
- **6-7/8 通过**：环境基本支持，部分能力受限。根据失败项给出具体建议。
- **4-5/8 通过**：环境支持有限，建议排查失败项后再使用复杂 Skills。
- **0-3/8 通过**：环境严重受限，建议检查智能体平台配置。

### 失败项的补救建议

| 失败项 | 补救建议 |
|---|---|
| SKILL.md 加载 | 检查 SKILL.md 是否在正确的 skills 目录下，检查 frontmatter 格式 |
| 环境变量读取 | 检查智能体平台是否支持 `metadata.openclaw.env` 声明，手动设置环境变量 |
| 文件写入 | 检查工作目录权限，确认挂载卷是否可写 |
| 文件读取 | 检查文件是否真的写入成功，检查路径是否正确 |
| 命令执行 | 检查容器是否禁用了 shell，检查用户权限 |
| 多步骤执行 | 检查智能体的上下文窗口是否足够，检查指令遵循能力 |
| 清理测试文件 | 检查删除权限，可能需要挂载卷支持 |
| 输出规范化 | 检查智能体是否支持严格的输出格式约束 |
